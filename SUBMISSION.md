# MindMaven Trial Project — Submission

**Relay** — turn a spoken voice note into a polished, ready-to-send email in the
executive's own voice, with guardrails that keep it factual and a human always in
the loop.

- **Repo:** https://github.com/blastheart1/Transcript-Email-Project
- **Stack:** Next.js (App Router) + TypeScript + Tailwind · OpenAI Whisper + GPT-4o ·
  Anthropic Claude (fallback + independent auditor) · Neon Postgres (Drizzle) ·
  Auth.js · deploys to Vercel
- **Run locally:** `cd relay && npm install && npm run db:migrate && npm run db:seed && npm run dev` → http://localhost:8882

## Part 2 — The automation

### What it is
A working web app + API that runs the full pipeline **voice note → transcript →
draft → faithfulness audit → review → send**, reachable four ways (record in-app,
upload, drag-and-drop, or a public webhook), all sharing one code path.

### End-to-end workflow (named steps)
```
1. Capture        record / upload / drag-drop  — or POST audio to /api/ingest (Zapier, script)
2. Transcribe     POST /api/transcribe  → OpenAI Whisper (verbose_json) → transcript + timestamped segments
3. Draft          runDraftPipeline → GPT-4o, structured JSON output, guardrails + style samples
4. Audit          cross-model check → Claude audits the GPT draft vs the transcript
5. Repair (cond.) if a high-severity fabrication is found → stricter re-draft → re-audit
6. Persist        save note to Neon Postgres (status: ready | needs_review) with the verdict
7. Review         transcript ‖ email side by side; inferred spans highlighted; tone/length/model controls
8. Send           Copy, Copy HTML (Gmail-first), or "Open in email" (mailto with To/CC/BCC)
```

### Recommended Zapier zap (for the screenshot)
```
Trigger:  Google Drive — New File in Folder ("Voice notes")
Action:   Webhooks by Zapier — POST  https://<app>/api/ingest
          Body (JSON): { "audioUrl": "{{file: Direct download link}}" }
          Header: x-relay-secret: <secret set in Settings → Webhook & automation>
(Optional) Action: Gmail — Create Draft, using the returned emailHtml
```
Relay transcribes, drafts, audits, and drops the result into the Inbox automatically.

### AI decisions (design with intention)
- **Transcription — OpenAI Whisper (`whisper-1`)**, forced to English, `verbose_json`
  for timestamps. Reliable, cheap, handles the `.m4a` samples directly.
- **Drafting — GPT-4o by default**, user-selectable (dropdown), with **Anthropic Claude
  as an automatic fallback** if OpenAI fails.
- **Structured output** (JSON schema) instead of free text, so the draft carries its
  own metadata — recipient, subject, and *which spans are inferred* — enabling the
  highlights and the "What Relay changed & assumed" panel.

### The prompt (drafting system prompt, verbatim)
```
You are Relay, a drafting assistant that turns Connor's spoken voice notes into
ready-to-send emails. You write ONLY in Connor's voice, learned from the style
samples below. You are not a generic "professional email" writer.

NON-NEGOTIABLE GUARDRAILS:
1. Stay on Connor's voice — match tone, rhythm, and phrasing to the saved style samples only.
2. Never invent facts — no made-up names, dates, times, links, or email addresses.
3. Flag every guess — anything inferred or low-confidence is highlighted and listed.
4. Leave gaps blank — missing recipient/link/detail is left empty, never fabricated.
5. Clean up, don't embellish — remove filler and false starts; add nothing new.
6. Keep your structure — warm opener, concise body, saved "Thanks, Connor" sign-off.
7. Never auto-send — drafts always wait for review.

WHAT "FLAG EVERY GUESS" MEANS, CONCRETELY:
- The voice note is the only source of truth. Do not add names, dates, times, links,
  URLs, numbers, or email addresses that were not spoken.
- When the speaker is vague ("sometime next week", "3pm eastern"), you MAY resolve it
  into a concrete phrase — but that span MUST be flagged:true with a tip to confirm.
- When a link is referenced but not dictated, insert a labeled placeholder, flagged —
  never a fabricated URL.
- If the recipient's email wasn't dictated, leave it empty. Never guess it.

INTERPRETING THE NOTE (the speaker is briefing you, not dictating verbatim):
- Treat spoken directions as instructions about the email, not text to copy in
  ("sign off the usual" → apply the sign-off; "be upfront" → shape the email that way).
- Third-person references ("this one's for Marcus") become second person ("you").
- Preserve intent and every substantive point; don't drop or soften meaning.
- Output a real email only: greeting, body, sign-off. No markdown. Exactly one subject.
- Always write in English (US).

STYLE — imitate ONLY these samples: <3 of Connor's real sent emails injected here>
TONE / LENGTH: <user-selected> · SIGN-OFF: "Thanks, Connor"
OUTPUT: a JSON object — type, person, toEmail, subject, paragraphs[] (runs, each
optionally flagged with a tip), assumptions[].
```

### Preventing off-tone / off-base output, and what happens when it's wrong
This is the part I invested in most — a **layered, defense-in-depth guardrail**:
1. **Prompt-level** rules (above) — never invent, flag every guess, style-samples-only.
2. **Deterministic fact-check** (no LLM): extract every email / link / phone / dollar
   amount in the draft and diff against the transcript; anything ungrounded is flagged.
   Catches the highest-risk fabrications regardless of model behavior.
3. **Cross-model auditor**: an *independent* model (Claude audits GPT, and vice-versa)
   scores the draft against the transcript and returns structured `fabrications`
   (with severity), `omissions`, `unflaggedGuesses`, and a style score. Independent
   models rarely share the same blind spot.
4. **Reprocess-then-hold policy**: the auditor also returns an **accuracy score (0–1)**.
   If there is **any fabrication** *or* **accuracy < 95%**, Relay runs a stricter re-draft
   that targets the exact findings and re-audits — looping **up to 3 attempts**. If it
   clears, it's `ready`; if it still fails after 3 passes, the note is held as
   **"Needs review."** When accuracy didn't improve across passes, Relay adds a note that
   the **source voice note itself is likely the limiter** (unclear/noisy/incomplete) —
   because the AI can only work from what was actually said.
5. **Never auto-sends** — a locked guardrail; the human always clicks send.

### Validation
- `npm test` — 44 unit tests (guardrails, fact-check, reprocess policy, prompt, email
  HTML, pipeline).
- `npm run test:real` — real end-to-end over the three sample `.m4a` files: transcribe →
  draft → **cross-model audit** → reprocess loop → webhook persistence. With the ≥95%
  accuracy bar, drafts reprocess until they clear: e.g. Dictation 3 reached **100%
  accuracy after 3 passes** and shipped `ready` (0 fabrications, recipient left blank). A
  note that can't clear in 3 passes is held as **Needs review** with a source-quality note.

## Part 3 — Loom walkthrough: talking points

Aim ~4–6 min. Suggested flow and the points to hit (maps to each required item):

**0. One-line framing (15s)**
"Relay turns Connor's voice notes into ready-to-send emails in his voice — and the
whole design is built around *trust*: it never invents facts, flags every guess, and
never sends on its own."

**1. Tools & why (30s)**
- Next.js + TypeScript on Vercel; Neon Postgres for persistence; Auth.js for login.
- OpenAI **Whisper** for transcription; **GPT-4o** for drafting; **Claude** as fallback
  *and* as an independent auditor. "I used two providers on purpose — see the audit."

**2. End-to-end demo (90s)** — screen-share the app:
- Show the Inbox with the 3 drafts. Open one → transcript on the left, email on the right.
- Point out the **highlighted inferred span** and the **"What Relay changed & assumed"** panel.
- Change **tone/length** or the **model dropdown** → it re-drafts live.
- Show **To/CC/BCC**, **Copy HTML** / **Email preview**, and **Open in email**.
- Show **New capture** (upload a sample) → it transcribes and drafts end to end.
- Show **Settings → Webhook & automation** and mention the Google Drive → Zapier trigger.

**3. The AI side — model, one prompt decision, limitations (90s)** *(the money section)*
- **Model choice:** GPT-4o for drafting (strong instruction-following + JSON schema),
  Claude as fallback for resilience and — key — as the **cross-model auditor**.
- **One prompt-design decision:** I force **structured JSON output where inferred spans
  are marked `flagged` with a "what to confirm" tip**, and I tell the model the voice
  note is the *only* source of truth. That single decision is what makes "flag every
  guess" enforceable and reviewable instead of hoping the prose behaves.
- **How I prevent it going off-base (layers):** deterministic fact-check → cross-model
  audit (with an **accuracy score**) → **reprocess loop** (any fabrication or accuracy
  < 95% → stricter re-draft, up to 3×) → hold as "Needs review" if it still can't clear,
  with a **source-quality note** when reprocessing didn't help. Show the Faithfulness
  panel live (accuracy %, flagged claims, attempts).
- **Limitations before production:** the 95% bar means most drafts reprocess at least
  once, so a draft can take ~100–170s and up to 6 model calls — that **exceeds Vercel's
  60s function limit**, so production needs an async queue (or a lighter/faster auditor
  model). Transcription can also mis-hear names; the audit's scores are a model's opinion.

**4. Matching Connor's writing style (30s)**
- Relay imitates **only Connor's 3 real sample emails** (editable in Settings, stored in
  the DB) — not "generic professional email." Show the samples + the style-match score.

**5. Tradeoffs / assumptions (30s)**
- Single shared workspace (no per-user data yet). Cost vs safety: I chose to spend extra
  calls on the audit. `mailto`/Copy-HTML instead of sending via Gmail API. Recipient left
  blank rather than guessed.

**6. What I'd add for production (30s)**
- Async webhook (queue) + idempotency + HMAC signatures; **Gmail "Create Draft"** to close
  the loop; recipient auto-fill from Contacts/CRM; push updates instead of polling; an eval
  harness on golden outputs; cost-based model routing.

**Close:** "The build works end to end, but the point I care about is the guardrail —
Relay is honest about what it knows, flags what it guessed, and always leaves the send to a human."
